最长回文子串 Manacher 算法

April 18, 2016 •  🍻 3min 47s read
🔖 算法字符串回文串manacher

算法简介

¶

Manacher 算法能够在线性时间内求出一个字符串中所有以任意位置或相邻的两个位置为中心的最长回文串。

算法原理

¶

记给定的文本字符串为 S，其长度为 N。为方便描述，在 S 中的任一对相邻字符间插入一个特殊字符 $，目的是保证新串中所有的回文子串长度都为奇数（实际实现时并不需要此操作）。设以 S_i 为中心的最长回文串长度为 2 r_i + 1。那么，如果 j < i 且 j + r_j > i，则作 i 点关于 j 的对称点 i'，则必有 i' > j - r_j。

情况一：i' - r_{i'} > j - r_j

¶

如下图所示，由对称性可以断言 r_i = r_{i'}。

事实上，假设 k' = i' - r_{i'} - 1 关于 j 的对称点为 k; k 关于 i 的对称点为 k_i；k_i 关于 j 的对称点为 k'_i 。如下图所示，显然有 S_k = S_{k'},\; S_{k_i}=S_{k'_i}，且 S_{k'} \neq S_{k'_i}；所以 S_k \neq S_{k_i}，故 r_i=r_{i'}。

情况二：i' - r_{i'} < j - r_j

¶

如下图所示，由对称性可以断言 r_i = j + r_j - i。

当 j + r_j \leqslant i 时，不能做出更多的假设，只能暴力匹配。幸运地是，可以证明暴力匹配的次数为总字符数，即 O(N) 的。

算法实现

¶

所谓的插入特殊字符是为了方便讲解，实现时可以将回文串 [l,r] 的长度存在 r_{l+r} 中。不难发现，当回文串长度为偶数时，l+r \equiv 1\mod(2)。

参考了《ACM国际大学生程序设计竞赛:算法与实现》的代码。

    void manacher(char* S, int* R, int n) {
      R[0] = 1;
      int _size = (n << 1) - 1;
      for (int i = 1, j = 0; i < _size; ++i) {
        int l = i >> 1, r = i - l;
        int rst = (j - 1 >> 1) + R[j];
        R[i] = rst < r ? 0 : min(rst - r + 1, R[(j << 1) - i]);
    

        while (
          l - R[i] >= 0 && 
          r + R[i] < n && 
          S[l - R[i]] == S[r + R[i]]
        ) ++R[i];
        if (r + R[i] > rst) j = i;
      }
    }

    function manacher(text: string, R: number[], N: number): void {
      R[0] = 1
      const _size = (N << 1) - 1 
      for (let i = 1, j = 0; i < _size; ++i) {
        const p: number = i >> 1
        const q: number = i - p
        const r: number = ((j + 1) >> 1) + R[j] - 1
    

        let L = r < q ? 0 : Math.min(r - q + 1, R[(j << 1) - i])
        while (p > L - 1 && q + L < N && text[p - L] === text[q + L]) L += 1
        R[i] = L
    

        // Update j
        if (q + L - 1 > r) j = i
      }
    }

复杂度分析

¶

由于每一次只从未被匹配过的字符出发往右扩展，一共只有 O(N) 个字符，所以复杂度是线性的。

  • 空间复杂度 O(N)
  • 时间复杂度 O(N)
