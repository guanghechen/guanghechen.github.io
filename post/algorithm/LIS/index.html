最长上升子序列（LIS）

June 02, 2021 •  🍻 12min 49s read
🔖 算法最长上升子序列LIS

对于数列 A=\lbrace a_1,a_2,\cdots,a_N \rbrace，其任一上升子序列 B=\lbrace a_{b_1}, a_{b_2}, \cdots, a_{b_M}\rbrace 满足 b_i < b_{i+1} 且 a_{b_i} < a_{b_{i+1}}。A 的最长上升子序列指 A 的所有上升子序列中元素个数最多的那些。

A 的字典序最小的最长上升子序列指 A 的最长上升子序列中（不妨设元素个数为 M），元素下标组成的 M 维向量最小的那个。

算法一

¶

找到 LIS 的长度

¶

记 dp_{i} 表示前 i 个数中以第 i 个数结尾的最长上升子序列长度，则状态转移方程为：

dp_{i}=\max \lbrace dp_x | 1 \leqslant x < i, a_x < a_{i} \rbrace + 1

function findLengthOfLIS(a: number[], N: number): number {
  const dp: number[] = new Array(N).fill(1)
  for (let i = 0; i < N; ++i) {
    let x = 0
    for (let j = 0; j < i; ++j) {
      if (a[j] < a[i] && x < dp[j]) x = dp[j]
    }
    dp[i] = x + 1
  }
  return dp[N - 1]
}

找到字典序最小的 LIS

¶

在完成上一步的运算后，维护一个数组 C=\lbrace c_1, c_2, \cdots, c_M\rbrace，其中，c_i 表示长度为 i 的上升子序列的最后一个元素在 A 中的最小下标，则字典序最小的 LIS 为：a_{c_1}, a_{c_2}, \cdots, a_{c_M}。

维护这个数组的复杂度为 O(N)，扫描到 i 时，若此时 c_{dp_i} 还未设置值，则将 c_{dp_i} 设置成 i，否则忽略它:

function findMinimumLexicographicOrderLIS(a: number[], N: number): number[] {
  const dp: number[] = new Array(N)
  const c: number[] = new Array(N).fill(-1)
  for (let i = 0; i < N; ++i) {
    let x = 0
    for (let j = 0; j < i; ++j) {
      if (a[j] < a[i] && x < dp[j]) x = dp[j]
    }
    dp[i] = x + 1
    if (c[x] === -1) c[x] = i
  }
  return c.slice(0, dp[N-1])
}

复杂度分析

¶

边界情况是找不到这样的 x，此时相当于 dp_i=1。不难发现，对于每个位置 i，需要扫描 i-1 个元素，因此

  • 时间复杂度：\displaystyle O\left(\sum_{i=1} ^ {N} (i - 1)\right)=O(N^2)

  • 空间复杂度：O(N)

算法二

¶

找到 LIS 的长度

¶

维护一个单调栈 S，扫描数列 A，若 a_i 大于栈顶元素，则压栈，否则替换掉 S 中第一个大于等于 a_i 的元素 S_x。则扫描完毕后，S 长度即为 A 的最长上升子序列的长度。因为 S 是单调栈，所以可以使用二分搜索 (lower_bound) 快速找到 S_x。

function findLengthOfLIS(a: number[], N: number): number {
  const s: number[] = [a[0]]
  for (let i = 1; i < N; ++i) {
    const y = a[i]
    const x = lowerBound(0, s.length, x => s[x] < y)


    // In Javascript / Typescript, the following condition can be omitted
    if (x > s.length) s.push(y)
    else s[x] = y
  }
  return s.length
}

¶

function findMinimumLexicographicOrderLIS(a: number[], N: number): number[] {
  const s: number[] = []
  const c: number[] = new Array(N).fill(-1)
  for (let i = 0; i < N; ++i) {
    const y = a[i]
    const x = lowerBound(0, s.length, x => s[x] < y)


    // In fact, we don’t really need the dp array, as we only need to know 
    // the length of the LIS formed by the first (i+1) elements, and that 
    // value is (x+1).
    if (c[x] === -1) c[x] = i


    // In Javascript / Typescript, the following condition can be omitted
    if (x > s.length) s.push(y)
    else s[x] = y
  }
  return c.slice(0, s.length)
}

复杂度分析

¶

扫描数列复杂度为 O(N)，每扫描到一个位置时，通过 O(\log N) 的二分搜索算法找到目标位置，故

  • 时间复杂度：O(N \log N)
  • 空间复杂度：O(N)

扩展

¶

上面提到的两个算法除了求 LCS 外，还都支持求 最长非严格单调递增子序列。

  • 算法一（O(N^2)），只要把转移方程改成如下形式就好了：

    dp_{i}=\max \lbrace dp_x | 1 \leqslant x < i, a_x \leqslant a_{i} \rbrace + 1

  • 算法二（O(N\log N)），只要在计算 lowerBound 时，把 s[x] < y 改成 s[x] <= y 就好了：
